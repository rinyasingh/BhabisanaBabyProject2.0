Web Portal

Providing an interface for Bhabhisana staff to create and assign ECD Content to their Parents.

Dashboard Interface for Bhabhisana Staff

Introduction and Context

The Bhabhisana staff had an excited reaction to the web portal being accessible through the web browser, opposed to a local application. "We are constantly on the move" highlights the convenience this portal provides by being accessible remotely. This web portal was built on a WordPress base with custom functionality and features developed through utilizing Pods, Oxygen Builder, and developing a custom plugin whcih used HTML, PHP and CSS. Below is a high level overview of Web Portal with relation to the implemented system.



The key aims of the web portal were as follows:

  • Create ECD Content Posts.
  • Create User (Parent) Profiles.
  • Manage and Filter ECD Content per Category and Language.
  • Assign Users to ECD Content.
  • Custom REST API Endpoint for Mobile Application to retrieve data.

Methodology

As highlighted within the home page, a co-design scrum approach was our project management. Sessions conducted with Bhabhisana staff included workshops, focus groups, and user (task) evaluations. A high level overview of our timeline and utilization of co-design sessions (and iterations) can be found here.

Create User Profiles and ECD Content

The creation and management of pages, posts and users is a native functionality in WordPress. Pods was used to create a Custom Post Type (CPT) called "ECD Content". The further customization of ECD Content came in the form of adding a custom field, "Assigned Users", to the CPT. Likewise, there was a custom field, "Assigned Posts", added to the User post-type. Thereafter, native categories and tags where used, where functionality was modified to suit it towards our requirements. The non functional requirement of scalability is easily managed within the WordPress ecosystem, additional ECD Content and Users in large sets is managed well. Error handling, input validation, and visibility of system status is well addressed within the CMS backend - much to the satisfaction of Bhabhisana staff, highlighting the robust usage of the system.

Assignment of Users to ECD Content

The "Assigned Posts" custom field displays an interface to assign ECD posts to a User. The "Assigned Users" custom field displays an interface to assign Userâ€™s to an ECD Content post. A bi-directional relationship exists between these fields. This means staff can perform the same task through either interface with no loss of data, which highlights the reliability and integrity of the system. The input of our custom fields, "Assigned Posts" and "Assigned User", are stored as metadata within wp_usermeta and wp_postmeta respectively. The mobile app fetches this information from the custom REST API endpoint that was created through the developed plugin.

Custom Plugin to create REST API Endpoint

Two additional endpoints were created for the mobile app to fetch this data. One endpoint was used for basic user authentication for the mobile app. This endpoint exists at ecd/v1/.../... where the input is validated within the database to ensure the username exists. The endpoint to retrieve assigned content for a user is at ecd/v1/.../. This endpoint merges the public content, which is all ECD Content assigned to the "Public" User, and additional ECD Content assigned to the provided username. Only English content is displayed within the app, so only English content is made available to fetch. This functionality was implemented through a custom plugin, and fulfills the key non-functional requirement of interoperability.

Dashboard (Portal) Customization

The custom plugin abstracts all unnecessary WordPress options and adds user help with quick links and online documentation. This allows for a streamlined dashboard where the Bhabhisana staff have no access over anything besides ECD Content and User management. Additionally to using a learnable and fun system, the portal is quick with executing key functions within the browser.

Results

Bhabhisana staff acknowledged "I am not technically advanced", but "I can get used to this and will gain confidence", which highlights the positive mindset they had to the usability of the system. On conclusion of the sessions with Bhabhisana staff, a usability evaluation was conducted to highlight the usability and satisfaction of the web portal.
