You and Your Baby Mobile Application

Co-Design

Co-designed with community members at the Bhabhisana Baby Project and at the HSRC in Sweetwaters, KZN
Community members decided on the functional and non-functional requirements
Community members gave input on design elements such as colors and choice of images

Iteration 1

The first iteration contained unprocessed ECD web portal content, pdf article viewer, and a facilities map search. The mobile application retrieved post content via a SQL connection to the web portal's database. The content was streamed from the hosted media files and was displayed through the native Android video view. Users were able to view a base set of PDF articles. The facilities search feature was initially implemented as an activity where users would create their own maps, placing markers on locations for later viewing.

Iteration 1 based on CoMaCH co-design activities

Iteration 2

The second iteration added the capabilities to download content and save media files to local storage. This ensured that the app remained usable during limited internet connectivity. The web portal content fetched was also pre-processed in this iteration to ensure that content was presented in an interpretable format. The search feature was implemented; however, it only returned exact query matches. The facilities search was refined to show users a list of curated local amenities and contact details of support structures across Western Cape. The creation of user maps was removed as it was quite cumbersome and didn't meet the requirements of the intended functionality. User authentication was added to this iteration and allowed users to login and authenticate themselves. It allowed users to sync/download content related to their profile. The user interface was updated to be fully responsive in portrait and landscape orientations. The design was improved making it visually appealing and aligned with the BBP branding.

Iteration 3

In the third iteration filtering and searching of data was improved significantly. Users were now able to filter posts by assigned category. The search functionality no longer required exact matches. Communication of system status was improved through the use of snackbar notifications which explain changes that occur in the system. Contextual activity-based help was implemented to ensure that users were able to easily recover from erroneous/unfavorable states.

Deployment

The application was deployed for official use at the end of September. It was well received by the users. Usage data shows regular use by community members. The application has been subject to maintenance and bug fixes after deployment as part of the ongoing co-design process.
