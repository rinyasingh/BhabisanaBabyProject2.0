Web App

Providing an interface for Bhabhisana parents to login, or guests to visit, and view their ECD Content. A web portal allowing the Bhabhisana Baby Project staff to upload content, manage users and modify these settings.


bbp-1.cs.uct.ac.za

Introduction and Context

The previous web system was developed using WordPress. WordPress allows users to create, manage and publish digital content such as text, images and videos.

The key features of the system were:

  • The staff-facing web portal performed the functionality of allowing web portal users to create and manage content and users, emphasising content assignment functionality.
  • The user-facing web application allows users to authenticate, view content based on categories and languages, and access assigned content. The mobile application was designed to have responsiveness, usability, and coherent branding features.

The problems identified with the 2022 deployment:

  • The staff reported that it was difficult to use the web portal due to its unintuitive interface.
  • The quality of JSON response returned by the WordPress API endpoints was large and poorly structured due to the lack of customisability that WordPress allows for API design. String manipulation needed to be done by the mobile application to extract file links.
  • There was a lack of proper user authentication. Users were only authenticated based on whether the username existed. When content was retrieved for the mobile application, only the user ID was used. There was no password check or session validation performed.

Results

The new system was built using the Django framework, featuring a Model-View-Template (MVT) pattern and Object-Relational Mapping (ORM) layer. The system's architecture is divided into apps, each with specific functionality, and the user interface is created using HTML templates and Bootstrap. The system emphasizes user authentication, security, and database design, using a PostgreSQL database and Django's ORM. The system is currently hosted on a Unix server with Apache and Docker containers.

The usability tests conducted throughout the project confirmed that the primary design objective of creating a user-centric system was achieved. Feedback from BBP staff participating in these tests indicated that the new web portal is a significant improvement over the previous one. They described it as more intuitive, with a cleaner and simpler interface, and one staff member even praised its appearance and ease of use.
